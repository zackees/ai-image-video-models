# ai-image-video-models

Holds AI models used in image/video upscaling, quality improvements and others.

Current assets live under `assets/` as zstd-compressed ONNX archives tracked by
Git LFS, indexed by an auto-generated manifest tree. The legacy `.pth` / `.zst`
files at the repo root are kept for older forks.

https://zackees.github.io/ai-image-video-models

## Layout

Three-level manifest tree, regenerated by `tools/gen_manifest.py`:

```
manifest.json                                  # root: {"categories": [{name, manifest_path}]}
assets/<category>/manifest.json                # category: {"models": [{name, manifest_path}]}
assets/<category>/<kind>/<model>/manifest.json # leaf: {"latest": "<ver>", "<ver>": {...}}
assets/<category>/<kind>/<model>/<model>-v<ver>.<ext>.zst   # LFS-tracked archive
assets/<category>/<kind>/<model>/notes.json    # optional provenance (see below)
```

Categories today: `qualityscaler/onnx` (9 fp16 ONNX models) and
`fluidframes/rife` (3 fp32 ONNX models).

## Consumer contract

Known consumers resolve models like this:

1. GET `https://raw.githubusercontent.com/zackees/ai-image-video-models/main/assets/<category>/<kind>/<model_key>/manifest.json`
   where `model_key = filename.removesuffix("_fp16.onnx" | "_fp32.onnx")`.
2. `entry = manifest[manifest["latest"]]`
3. Download `entry["href"]` (a `media.githubusercontent.com` URL that serves the
   raw zstd bytes from LFS — never an LFS pointer).
4. Stream-sha256 the download against `entry["sha256"]`; abort on mismatch.
5. Zstd-decompress to `<model_path>.tmp`, then atomic `os.replace`.

Every leaf version entry MUST therefore have:

| field | type | constraint |
|-------|------|-----------|
| `href` | str | https URL serving raw zstd bytes |
| `sha256` | str | 64-char lowercase hex of the archive |
| `size` | int | exact byte size of the archive |
| `compression` | str | must be `"zstd"` (the only value consumers handle) |
| `notes` | str | optional, ignored by consumers |

## Adding a new model

1. Drop the archive into `assets/<category>/<kind>/<model>/<model>-v<version>.<ext>.zst`
   (zstd level 19; the `-v<version>` part is required).
2. Optionally create/update `notes.json` in the same dir:
   `{"default": "text", "<version>": "version-specific text"}`.
3. Run `python tools/gen_manifest.py` and commit the regenerated manifests.

CI blocks the PR if you skip the regen (`git diff --exit-code` on manifests) and
validates every manifest field against the LFS pointer sha256/size. A weekly
network probe range-GETs every `href` and asserts the zstd magic bytes, catching
LFS-pointer leaks at the media endpoint.
